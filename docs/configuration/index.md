## lime-* files

The LibreMesh configuration is mainly done by a set of `/etc/config/lime-*` files.

::: tip
These files can be managed via UCI https://openwrt.org/docs/guide-user/base-system/uci
:::

Here from the lowest to the highest priority, **each one overrides the previous one**:

``` sh-vue
/etc/config/
├─ lime-defaults - the default configuration
├─ lime-community - community specific configuration
├─ lime-<MAC> - node specific configuration, each node reads only it's own
├─ lime-node - node specific configuration
└─ lime-autogen - DO NOT EDIT, autogenerated configuration
```
::: warning
Interface specific options have to be included in `lime-node` or `lime-<MAC>`
if in **lime-community** they'll cause unpredictable behaviour.
:::


### Parametrizable options
Inside `lime-* files` the options marked with "Parametrizable with %Mn, %Nn, %H", can include the templates:
- `%Mn`: replaced by the n'th byte of the **primary_interface MAC**
- `%Nn`: replaced by the n'th (n = 1..5) network-identifier byte, calculated from the **hash of the ap_ssid**,
 	   so that all the nodes that form a mesh cloud (share the same ap_ssid) will produce the same value
- `%H`: replaced by the **hostname**
